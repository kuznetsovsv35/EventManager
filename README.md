# Event Manager Web API application
Продвинутая разработка на C# и .NET. Проектная работа.

# Работа над проектом.
1. Создан каркас приложения (базовая структура).
2. Создание модели событияю.
3. Реализация хранилища события в памяти на основе TF Core.
4. Реализация интерфейса IEventService и объектов DTO.
5. Реализция EventsControler (API контролера).
6. Реализация атрибута валидации класса модели входящих данных.
7. Подготвка к публикации проекта (мелкие правки и форматирование,, документирование кода).

# Сборка и запуск проекта.
1. Клонирование репозитория:
   ```bash
   git clone -b sprint-1 git@github.com:kuznetsovsv35/EventManager.git sprint-1
2. Перейти в паку "sprint-1":
   ```bash
   cd sprint-1
3. Сборка проекта (целевой фреймворк net10.0):
   ```bash
   dotnet build
4. запуск приложения:
   ```bash
   dotnet run
5. В браузере открыть: http://localhost:5108/swagger
6. На странице можно увидеть описание API и попробовать его в деле.

# Описание API.
Таблица: Методы API, данные запросов/ответов, возможные ответы.
| Метод | URI | Действие | Данные запроса | Ответ | Данные ответа | Возможные ошибки |
|-------|-----|----------|----------------|-------|---------------|------------------|
|GET|/events|Получить все события.|Нет|OK|JSON-массив объектов данных событий|
|GET|/events/{id:uuid}|Получить событие по идентификатору id.|id - ID события|OK|JSON-объект-описание события|NOT_FOUND|
|POST|/events|Создать событие.|JSON-объект информации о событии|CREATED|JSON-объект-описание события|BAD_REQUEST|
|PUT|/events/{id:uuid}|Обновить данные о событии.|ID обытия, JSON-объект информации о событии|OK|JSON-объект-описание события|BAD_REQUEST, NOT_FOUND|
|DELETE|/events/{id:guid}|Удалить событие по идентификатору id.|id - ID удаляемого события|OK|Данные удаленного события|NOT_FOUND|

1. GET: /Events - Получить все события.
  Ответы: OK (массив данных событий 
2. GET: /Events/{id:guid} - Получить событие по идентификатору id.
  Ответы: OK (тело: данные) или NOT_FOUND.
3. POST: /Events - Создать событие. Тело запроса JSON-объект, согласно схемы.
  Ответы: Created (тело: созданный объект события).
4. PUT: /Events/{id:guid} - Обновить данные события по идентификатору id, Тело запроса JSON-объект, согласно схемы. 
  Ответы: OK (тело: данные обновленного события) или NOT_FOUND.
5. DELETE /Events/{id:guid} - Удалить событие по идентификатору id.
  Ответы: OK (данные удаленного события) или NOT_FOUND.
